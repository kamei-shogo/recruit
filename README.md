## GitHub Projects のセットアップ

### 1. プロジェクトを作成する

1. このリポジトリのページで **Projects** タブ → **Link a project** → **New project** をクリック
2. **Table** ビューを選択し、プロジェクト名（例: `就活管理 2026`）を入力して作成

### 2. カスタムフィールドを追加する

テーブルの右端にある **`+`** ボタンからフィールドを追加します。推奨フィールドは以下の通りです。

| フィールド名 | 種類 | 設定例 |
|---|---|---|
| `企業名` | Title（デフォルト） | — |
| `業種` | Single select | IT / 製造 / 金融 / コンサル / 商社 / その他 |
| `年収（万円）` | Number | — |
| `ステータス` | Single select | 未エントリー / ES提出済 / 書類選考中 / 面接中 / 内定 / 不合格 / 辞退 |
| `締め切り日` | Date | — |
| `ESリンク` | Text | リポジトリ内のESファイルへのパス |
| `企業URL` | Text | 採用ページURL |
| `メモ` | Text | 自由記述 |

### 3. ビューを追加する

- **テーブルビュー**: 全企業を一覧表示（デフォルト）
- **カンバンビュー**: `ステータス` フィールドでグループ化し、選考ステップを視覚的に管理
  - **Board** ビューを追加 → **Group by** → `ステータス` を選択
- **フィルタービュー**: 例）「今週締め切り」フィルター
  - `締め切り日` が今週内 の条件で絞り込んで保存

### 4. Issue と紐付ける

企業ごとに Issue を作成してプロジェクトに追加すると、コメントや添付で選考メモを一元管理できます。

1. Issue テンプレート（`.github/ISSUE_TEMPLATE/company.yml`）を使って企業 Issue を作成
2. Issue の右サイドバー **Projects** からプロジェクトに追加

## ESファイルの管理方法

### ファイルの命名規則

```
ES/<企業名>_<提出日 YYYYMMDD>.md
```

例:
```
ES/Sharp_20260312.md
ES/Toyota_20260401.md
```

### ESファイルの作成手順

1. `ES/_template.md` をコピーして企業名と日付でリネームします。
2. テンプレートに沿って内容を記入します。
3. コミット・プッシュしてリポジトリに保存します。

### GitHub Projects からリンクする

GitHub Projects のテーブルで `ESリンク` フィールドに相対パスを貼ります。

例:
```
ES/Sharp_20260312.md
```

GitHub 上でリンクをクリックすると、Markdown ファイルは整形された状態で即座に表示されます。

## 運用フロー

```
企業を発見
    ↓
companies/_template.md をコピーして企業情報ファイルを作成
    ↓
GitHub Projects のテーブルに行を追加（または Issue を作成して紐付け）
    ↓
ES を作成 → ES/_template.md をコピーして記入 → コミット
    ↓
Projects の ESリンク フィールドにファイルパスを記入
    ↓
ステータスを更新しながら選考を進める
```
